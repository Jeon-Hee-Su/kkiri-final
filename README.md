# 🤝 끼리 (KKIRI) - 모임 통합 관리 서비스

> 모임 생성부터 회비 관리, 정산, 자동이체까지 한 번에 해결하는 금융 플랫폼

<br>

## 📌 프로젝트 소개

**끼리(KKIRI)** 는 모임의 회비를 투명하게 관리하고, 정산과 자동이체를 자동화하여  
모임 운영의 불편함을 해소하는 모임 통합 관리 서비스입니다.

- 📅 개발 기간: 2026.02 ~ 2026.04
- 👥 팀 구성: 5인 팀 프로젝트
- 🙋 본인 역할: 프론트엔드 개발 및 백엔드 API 연동 일부

<br>

## ✅ 담당 기능 (전희수)

- 아이디 실시간 중복 확인 및 유효성 검증
- PortOne SDK 연동 실명/휴대폰 본인인증
- 비밀번호 조건 검증 및 약관 동의 상태 동기화 처리
- OAuth2 소셜 로그인 추가 정보 입력 화면 구성
- 로그인 시 사용자 인증 처리 및 권한 관리

### 👥 그룹 생성 / 모임 통장 개설
- 그룹 생성 화면 및 단계별 흐름 구현
- 모임 전용 가상 계좌 개설 화면 구성 (은행 선택, 약관 동의, 개설 완료)
- 결제 비밀번호 6자리 숫자 키패드 UI 직접 구현
- 방장 계좌 연결 화면 구성

### 📬 초대 카드
- QR 코드 생성 및 초대 코드 표시 화면 구현
- 이미지로 저장하기 기능 구현 (화면 캡처 후 다운로드)

### ⚙️ 그룹 설정 및 관리
- 그룹 정보 수정, 회비 규칙 설정, 미납자 자동 선택 UI 구현
- 그룹 정보 및 멤버 목록 조회 화면 구성 및 데이터 바인딩 처리
- 그룹 상세 조회, 회비 규칙 업데이트, 그룹 삭제 API 연동

### 📊 그룹 정산
- 지출 항목 선택 + 멤버 선택으로 1/N 금액 자동 계산 화면 구현
- 정산 결과 데이터 실시간 반영 처리

### 📜 거래 내역 조회
- 기간별 필터(1개월 / 3개월 / 직접 입력) 화면 구성
- 기간 설정 모달 구현

### 💳 결제 수단 관리
- 결제 수단 선택 화면 UI 구성
- 결제 비밀번호 입력 모달 공통 컴포넌트 구현

## 💡 트러블슈팅

### 1/N 정산 데이터 불일치 문제

**문제**  
정산 기능 테스트 중 특정 멤버의 정산 금액이 계산 결과와 다르게 표시되거나, 일부 멤버의 정산 상태가 화면에서 누락되는 현상 발생

**원인**  
프론트엔드에서 전달하는 JSON 데이터 구조와 백엔드 DTO 구조가 일치하지 않아 데이터 바인딩 과정에서 값 누락 및 오매핑 발생

**해결**  
- 프론트엔드와 백엔드 간 API 명세 전수 분석
- 백엔드 DTO 구조에 맞춰 프론트엔드 데이터 전송 객체 재설계
- 이후 기능 개발 전 팀원과 데이터 구조 및 API 스펙을 먼저 정의하는 협업 프로세스 정착

## 📂 프로젝트 구조

```
src/
├── main/
│   ├── java/com/kkiri/
│   │   ├── controller/     # REST API 컨트롤러
│   │   ├── service/        # 비즈니스 로직
│   │   ├── mapper/         # MyBatis Mapper
│   │   ├── vo/             # VO 클래스
│   │   ├── config/         # Security, JWT 설정
│   │   └── scheduler/      # 자동이체 스케줄러
│   └── resources/
│       ├── templates/      # Thymeleaf 템플릿
│       ├── static/         # CSS, JS, 이미지
│       └── mappers/        # MyBatis XML
python_scripts/             # AI 지출 분석 스크립트
```
